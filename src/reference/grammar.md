# قواعد النحو (Grammar)

نحوُ لغة ص موصوفٌ وصفًا رسميًّا **مدفوعًا بالبيانات** (data-driven): كلّ قاعدة إنتاج
مكتوبة بصيغة EBNF ومربوطة بدالة التحليل الفعليّة في المحلّل (حقل `maps_to`)، فيتحقّق
الـCI من تطابق الوصف مع السلوك. هذا يجعل النحو **مصدرَ حقيقة قابلًا للتنفيذ** لا مجرّد
توثيق.

> **المصدر:** [`language-truth/grammar/`](https://github.com/sadlang/s-programming-language/tree/sadlang/language-truth/grammar)
> — موزّع على ثماني طبقات.

## صيغة الترميز (EBNF)

تتبع القواعد ترميز EBNF القياسيّ:

- `{ X }` تكرار صفر أو أكثر.
- `[ X ]` اختياريّ.
- `X | Y` بديل.
- `( ... )` تجميع.
- الطرفيّات تشير إلى [المعجم](keywords.md) ولا تُعرَّف داخل ملفّات النحو.

## أمثلة من القواعد

البنية العليا للبرنامج والكتلة (الكتلة تُغلَق دائمًا بـ`نهاية`):

```ebnf
Program     = { Declaration } EOF ;
Block       = { Statement } "نهاية" ;
IfStmt      = "إذا" Expression Block [ "وإلا" Block ] ;
WhileStmt   = "بينما" Expression Block ;
ForStmt     = "لكل" Identifier "في" Expression Block ;
FuncDecl    = "دالة" Identifier "(" [ ParamList ] ")" [ ":" Type ] Block ;
MatchStmt   = "طابق" Expression { "عندما" Pattern Block } [ "افتراضي" Block ] "نهاية" ;
```

> هذه الأمثلة توضيحيّة للقارئ؛ الصيغة الرسميّة الكاملة (مع `alternatives` و`maps_to`
> و`ast_node` لكلّ قاعدة) في ملفّات `language-truth/grammar/`.

## مبادئ نحويّة ثابتة

- **كلّ كتلة تُغلَق بـ`نهاية`** (عدا كتل `باستخدام` التي تُغلَق بـ`نهاية_استخدام`).
- **الفواصل متكافئة:** `،` ≡ `,` و`؛` ≡ `;`.
- **التعليقات:** سطريّ `#`، كتليّ `#* ... *#`، توثيقيّ `##`.
- **الرموز العربيّة حصريّة** حيث نُصَّ عليها (مثل `؟` = U+061F لأمان العدم والثلاثيّ).

## التوثيق المُولَّد

تُولَّد صفحات قواعد المحلّل آليًّا من هذا المصدر إلى
`docs/parser_rule/_generated` في المستودع الأساسيّ عبر `gen_parser_grammar_docs.py`،
فيبقى التوثيق متزامنًا مع السلوك.
